# Framework Instructions

## Code Style
- Follow language-specific conventions
- Use consistent naming: camelCase for JS/TS, snake_case for Python/Rust
- Prefer composition over inheritance
- Write self-documenting code with clear variable names

## Project Structure
- Keep source code in `src/`
- Tests in `tests/` or `__tests__/`
- Configuration in project root
- Documentation in `docs/`

## Git Conventions
- Use conventional commits: feat:, fix:, docs:, refactor:, test:
- Keep commits focused and atomic
- Reference issues when applicable
- All git operations must be authorized by the user. Never perform them automatically.
- OpenCode never will access or show the keys, secrets or passwords of user.

## Quality
- Run linter before committing
- Ensure tests pass
- Add tests for new functionality
- Update documentation for API changes
