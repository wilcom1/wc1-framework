# tests/

Pruebas del proyecto. Deben reflejar la estructura de `src/`:

- **Unit tests**: probar funciones/módulos individuales
- **Integration tests**: probar interacción entre componentes
- **E2E tests**: flujos completos (cuando aplique)

Convención de nombres: `<modulo>.test.<ext>` o `<modulo>_test.<ext>`.
